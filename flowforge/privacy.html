Privacy Policy

Effective Date: March 7, 2026

Introduction

FlowForge ("the App") is a client-side, offline-first project management application. This Privacy Policy explains how we collect, use, and protect your data.

1. Data Storage & Location

FlowForge runs entirely in your web browser and stores all data locally using your browser's localStorage and sessionStorage. Your data never leaves your device unless you explicitly export it.

  • Workspaces, projects, boards, and tasks: Stored locally under versioned keys in localStorage.
  • User accounts & credentials: Email, password, and display name stored in localStorage (plain text; hashing recommended for production).
  • Authentication tokens: Two-factor secrets (OTP) stored locally; never transmitted.
  • Filters and preferences: Stored locally; persist across sessions.

2. Analytics

FlowForge collects offline analytics events to measure app usage and feature adoption. Events are stored locally in your browser and not sent to any server.

  • Events tracked: User login/logout, task creation/editing/deletion, board operations, filter usage, exports, and 2FA changes.
  • Sensitive data excluded: Passwords, OTP secrets, codes, and tokens are automatically filtered and never included in analytics.
  • User metadata: Your userId, username, and email are included in analytics for session tracking only.
  • Storage: Analytics events are kept in localStorage with a 5,000-event rolling window.

3. Authentication & Security

Authentication is simulated for demo purposes and stored in plain text.

  • Email & Password: Stored locally in plain text; in production, passwords should be hashed with bcrypt or similar.
  • Two-Factor Authentication (2FA): Uses TOTP (Time-based One-Time Password) with 10-byte Base32-encoded secrets. Secrets are stored locally and verified in the browser; never transmitted.
  • No server credential storage: Since FlowForge is client-side, there is no backend credential database.

4. Multi-User & Data Isolation

FlowForge supports multiple user accounts on the same browser. Each user's data is isolated under a unique storage key.

  • Data segregation: Each user's workspace is stored separately (e.g., `flowforge_state_v2_u-xxx`).
  • Switching users: The app switches storage keys when you log in or switch accounts; your previous user's data remains in localStorage.
  • Data migration: Anonymous workspace data is copied to a new user's account on first login to preserve work.

5. Data Deletion & Control

You have full control over your data:

  • Clear browser data: Open your browser's developer tools (DevTools) → Storage/Application tab → clear localStorage to delete all FlowForge data.
  • Delete specific user: Via the auth module (if exposed in UI), remove user records from localStorage.
  • Export data: Use the "Export Workspace" button to download your data as a JSON file for backup or migration.
  • Logout: Clears the active user session but does not delete historical data from localStorage.

6. Cookies & Tracking

FlowForge does not use cookies or any third-party tracking. Theme preference and user session state are stored in localStorage (not cookies).

7. Third-Party Services

FlowForge may use external services for optional features:

  • QR Code Generation: When enabling 2FA, the app generates QR codes using Google Charts API. A temporary image request is sent to Google's servers containing your provisioning URL.
  • No advertising or analytics integrations: FlowForge does not embed Google Analytics, Facebook Pixel, or other third-party trackers.

8. Data Retention

Data is retained until you delete it. FlowForge does not automatically purge old data.

  • Analytics events: Kept in localStorage with a 5,000-event rolling oldest-event-first window.
  • Task history & activity logs: Retained indefinitely unless manually deleted.

9. Privacy for Children

FlowForge is not intended for children under 13. We do not knowingly collect personal information from children.

10. Changes to This Policy

This Privacy Policy may be updated periodically. Changes will be noted with an updated effective date. Continued use of FlowForge constitutes acceptance of the updated policy.

11. Contact & Support

FlowForge is an open-source demo project. For questions about privacy or data handling, refer to the README or review the source code on GitHub.

FlowForge is a vanilla-JavaScript, offline-first project management tool with built-in authentication and analytics—all running locally in your browser.
